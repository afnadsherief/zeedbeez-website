# Folder Structure

## Principle
Organize by feature and shared system, not by file type alone.

## Example Structure
- app/
- components/
- features/
- lib/
- hooks/
- providers/
- services/
- shaders/
- store/
- styles/
- three/
- types/
- utils/
- docs/

## Rule
Reusable systems live in shared modules. Page-specific content stays in feature modules.
